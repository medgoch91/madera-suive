1. نظرة عامة

MADERA DECO Suivi App هي PWA (Progressive Web App) لإدارة الأعمال اليومية ل MADERA DECO (نجارة، توزيع كهربائي، فواتير) — تجمع المخزون، الفواتير، الشيكات، الصندوق، الخدامة (الأجراء + PCs + التقنيون)، و إعلامات Telegram ف نظام واحد.

أهم الميزات (2026-05-09)

  • 📋 Bons d'entrée من الفورنيسورات مع صور إثبات (proof scan)
  • 📤 Bons de sortie منظمين par département (m:n article tagging)
  • 🧾 فواتير + ديفي (devis) سلسلتين رقم منفصلتين (DEV- vs FAC-)
  • 💳 شيكات + كمبيالات + نقد مع تنبيهات استحقاق
  • 💵 صندوق ledger كامل مع mirror تلقائي للحركات
  • 👥 أجراء أسبوعيين + 🟠 demi (نص نهار) + ⏱ heures supplémentaires + سلف
  • 👷 عمال PCs piece-work catalogue dédié
  • ⚡ تقنيون à distance avec bons sortie + recettes BOM
  • 📷 صور bons + cheques + factures (Supabase Storage)
  • 🤖 Telegram bot avec /today, /khlas, /caisse, /balance, /stock، notifs مباشرة
  • 📊 3 cron jobs نهارية: workers_eod, daily_report, caisse_eod
  • 🔐 Roles + permissions: admin, comptable, magasinier, commercial, technicien, viewer
  • 💾 نسخ احتياطية يومية Telegram + FTP (JSON dump 29 جدول)

3. صفحات التطبيق

📊 الداشبورد

الصفحة الرئيسية. KPIs دلّحظة دلوقت:

  • إجمالي البونات في الفترة المختارة
  • المبالغ المعلقة (شيكات بلا تسوية)
  • الصندوق الحالي + إيرادات/مصاريف اليوم
  • الخدامة (أجراء + PCs + تقنيون) مع KPI per-bucket
  • التنبيهات (شيكات تستحق اليوم، مخزون منخفض، إلخ)

📦 فورنيسور و السلعة (Catalog Hub)

3 tabs ف نفس الصفحة:

Tabالمحتوى
📦 السلعكتالوغ المنتجات: نوم، unité، عائلة (cat)، ref، صورة، سعر par fournisseur (table prix), departments tags (max 3), المخزون + min limit
🏢 الفورنيسوراتقائمة المورّدين: ICE, IF, RC, tel, email, families, totaux dette/credit
📋 البوناتقائمة bons d'entrée + tab "+ بون جديد" لإنشاء واحد
📷 صور: كل سلعة قادر تشد صورة (localStorage). كل بون / شيك / فاتورة قادر يشد صورة دلورقة الأصلية f Storage bucket bon-photos (proof scan).

📤 بونات الخروج (Bons de Sortie)

Hub جديد — bons de sortie منظمين par département. Tabs ديناميكية مبنية من جدول departments (CRUD).

  • 📋 بون جديد: اختار قسم → article picker مفلتر par dept tag → multi-line + qty + stock validation per row
  • 📜 السجل: قائمة كل البونات السابقة + filters (date range / search / dept) + actions (✏️ edit / 🗑️ delete / 📨 photo-share / 📥 print)
  • ⚙️ Depts: modal CRUD لإضافة/تعديل/أرشفة الأقسام (code, name, icon, color, position)

الأقسام الموجودة: ⚡ Élec, MIROIR, EMBALLAGE, MONTAGE, FINITION (و قابل لإضافة جديدة).

تنسيق الرقم: BS-{CODE}-YYYYMMDD-NNN (مثال: BS-ELEC-20260509-001).

Print: template احترافي مع logo + watermark + signatures (li khraj / li khda) — same style كي bulletin de paie.

👥 الأجراء (الخدامة)

3 sub-tabs:

Tabالمحتوى
👥 الأجراء (workers)عمال أسبوعيين: pointage يومي (✅ present / 🟠 demi / 🔴 absent / 🟡 conge) + ⏱ heures supp + 🔧 إنتاج بالقطعة
👷 par PCsعمال piece-work: الانتاج اليومي per PC (e.g. Tiroir 80 × 13 DH) — كتالوغ مشترك (sal_catalogue) لتجنب double-entry
⚡ électricité à distanceتقنيون subcontracting (شوف §3.7)

Workflow التسوية الأسبوعية:

  1. اختر الأسبوع (Sun → Sat)
  2. دير pointage يومي (مع قيمة إضافية إن وجدت)
  3. كل عامل ف الـdashboard كيبان net = (أيام × سعر) + hsup − سلف
  4. اضغط "💰 خلاص" → تأكيد → automatic Telegram notif + PDF payslip
  5. إذا السلف > الأجر → automatic rollover للأسبوع الجاي
  6. 📄 / 📨 buttons لطبع/مشاركة payslip بدون settlement
ℹ️ معدلات تاريخية: salary_rates table بتدير effective-dated rate lookups — رفع الأجر اليوم ما كيرتراجعش على pointage الفترات السابقة.

💵 الصندوق

Cash-box ledger كامل. كل حركة (سلف، تسويات، payments) كي تكون auto-mirrored من جدولها الأصلي.

  • Manual entries (إيرادات / مصاريف غير مرتبطة)
  • Auto-mirror من salarie_taswiyas, pc_taswiyas, technician_payments, salarie_avances, pc_avances
  • كل حركة كتفير Telegram notif مع المتقدم الجاري دلصندوق
  • Daily summary auto à 21:00 Casa

🗒️ التسويات (شيكات + كمبيالات + نقد)

3 أنواع:

  • 💳 شيك (cheque): مع رقم + تاريخ إصدار + تاريخ استحقاق
  • 📜 كمبيالة (effet): نفس البنية
  • 💵 نقد (espèce): بدون استحقاق (مدفوع فوراً)

Statuses: 🟡 معلق / ✅ مدفوع / 🔴 مرتجع

تنبيهات:

  • 🌅 08:00: cheques_due_morning — قائمة الشيكات اللي تستحق اليوم
  • 🕓 16:00 + 18:00: cheques_today_ping — رسالة prompt per شيك مع inline buttons (✅ مدفوع / ❌ باقي / 📅 أجّل 7 أيام)

🧾 الفواتير + ديفي (Devis)

Both share the factures table مع serie column ('fact' | 'devis').

TypeFormatColor
DevisDEV-{YYYY}-{NNNN}🟣 بنفسجي
FactureFAC-{YYYY}-{NNNN}🔵 أزرق

كل serie عندو sequence مستقل per-year (DEV-2026-0001 + FAC-2026-0001 يقدرو كاينين فس نفس الوقت).

Workflow:

  1. إنشاء فاتورة جديدة مع checkbox "📝 احفظ كـ ديفي" → status = 'ديفي'
  2. إن قبل الزبون → اضغط "✅ حول لفاتورة" → re-numbered كـFAC-... + status 'معلقة'
  3. كل فاتورة قادر يشد صورة + photo-share + print A4 template

⚡ électricité à distance

Sub-contracting workflow للتقنيين:

  1. Envoi matière: أرسل المادة الأولية للتقني (ينقص من articles.stock + INSERT material_dispatches)
  2. Recettes (BOM): per produit fini، حدّد المواد الخام المطلوبة (Bill of Materials)
  3. Livraison produit: التقني يجيب المنتج الفيني → ينقص من رصيد التقني (dispatches − consommation recettes), ما يمسش articles.stock
  4. Soldes: per-tech balance view (تاخذ − استعمل = باقي)
  5. Payments: سجّل الدفع (auto Telegram + caisse mirror)

⚙️ إدارة المستخدمين

Admin-only page (مرئية ل role 'admin' فقط). إدارة Supabase Auth users:

  • إضافة user جديد (ميل + password)
  • تعيين role (admin, comptable, magasinier, commercial, technicien, viewer)
  • per-page permissions toggles (مثلا magasinier ما يقدرش يفتح factures)
  • reset password
  • audit log بكل العمليات

4. الـ Workflows الرئيسية

4.1 شراء سلعة من فورنيسور

  1. 📦 فورنيسور و السلعة → tab بونات → ➕ بون جديد
  2. اختار الفورنيسور + التاريخ
  3. زيد lignes (article + qte + prix)
  4. احفظ → stock يزيد automatique + Telegram notif
  5. 📷 (اختياري) شد صورة دلورقة الأصلية

4.2 خروج سلعة لورشة

  1. 📤 بونات الخروج → اختار القسم (e.g. ⚡ الكهرباء)
  2. 📋 بون جديد → اختار articles + qty (مفلترين par dept tag)
  3. زيد destination (chantier optional) + ملاحظات
  4. احفظ → stock ينقص automatique + Telegram notif مع الـlines
  5. 📨 photo-share أو 📥 طبع PDF

4.3 إصدار فاتورة

  1. 🧾 الفواتير → ➕ فاتورة جديدة
  2. اختار client + lignes + TVA
  3. اختياري: ☑️ "احفظ كـ ديفي" → DEV-...
  4. احفظ → Telegram notif
  5. 📷 شد صورة دلورقة (proof) إن طلبتها
  6. إذا ديفي → b3da اضغط "✅ حول لفاتورة" → DEV → FAC

4.4 خلاص شيك مستحق

طريقتين:

  1. من Telegram: الشيك كيوصل لي 16:00 و 18:00 مع buttons → اضغط "✅ مدفوع" — انتها
  2. من التطبيق: 🗒️ التسويات → اختار الشيك → ✏️ edit → status = 'مدفوع' → احفظ

4.5 خلاص أجير أسبوعي

  1. 👥 الأجراء → tab dashboard → اختار الأسبوع
  2. كل عامل كيبان: أيام، أجر، إنتاج، سلف، الصافي
  3. اضغط "💰 خلاص" → confirm
  4. Auto: caisse out + Telegram notif + PDF payslip image مع توقيعات

4.6 إدخال PC جديد لعامل

  1. 👥 الأجراء → tab par PCs → اختار العامل
  2. ف form إضافة PC: اكتب اسم PC ولا اختر من dropdown (catalog)
  3. اختياري: ☑️ "زيد فالكتالوغ" → باش يبان لعمال آخرين
  4. اكتب الثمن (auto-fill من catalog إذا اخترتو)
  5. احفظ

  • 📦 بون جديد / تعديل بون

  • 💳 شيك جديد / تعديل / دفع

  • 👤 pointage update (debounced 10 min) — coalesced into one message
  • 💰 تخلص (sal/pc/tech) — مع running total اليوم
  • ⬆️ ⬇️ كل حركة دلصندوق (in/out) مع solde جديد
  • ⚠️ مخزون منخفض (تحت stock_min) — مل BS save

9. النسخ الاحتياطي

Auto-backup يومي (02:00 Casa)

  • Telegram: sendDocument JSON dump (29 جدول) لكل chat ف bot_subscribers
  • FTP Hostinger: upload نفس JSON ل /backups/backup-maderadeco-YYYY-MM-DD.json
  • Google Drive: wired لكن disabled (يحتاج GCP service account)

Restore tool

👉 https://maderadeco.app/restore.html (login → upload backup JSON → preview row counts → checkbox + button أحمر → atomic restore via restore_data(jsonb) RPC)

⚠️ تحذير: Restore يدير TRUNCATE 3la كل الجداول قبل INSERT — تأكد ما عندكش بيانات جديدة لي ما خاصش تتمسح.
